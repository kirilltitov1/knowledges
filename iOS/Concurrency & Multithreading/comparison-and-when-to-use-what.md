# Сравнение подходов и когда что использовать

## Threads
- Плюсы: полный контроль
- Минусы: сложно, опасно, редко нужно
- Использовать: спец. кейсы с RunLoop/низкоуровневые интеграции

## GCD
- Плюсы: простой, быстрый, пул потоков, группы/семафоры/барьеры
- Минусы: низкий уровень абстракции, нет структурированной отмены
- Использовать: простые фоновые задачи, параллельные вычисления

## Operations
- Плюсы: зависимости, повторное использование, отмена/приоритеты
- Минусы: сложнее, boilerplate для AsyncOperation
- Использовать: сложная оркестрация работ

## Swift Concurrency
- Плюсы: читабельность, structured concurrency, акторы, интеграция с языком
- Минусы: требования к платформе, новые концепции (Sendable, isolation)
- Использовать: дефолтный выбор для нового кода

## Короткие рекомендации
- По умолчанию — Swift Concurrency.
- Нужны зависимости и повторное использование — Operation.
- Простые задачи — GCD.
- Threads — почти никогда.
