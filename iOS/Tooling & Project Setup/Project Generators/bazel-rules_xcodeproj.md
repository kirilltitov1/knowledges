# Bazel + rules_xcodeproj — генерация Xcode проекта поверх Bazel

## Зачем
- Быстрые и детерминированные сборки, кэширование CI, масштабирование монорепозитория.
- Генерация Xcode проекта для локальной разработки из Bazel‑графа.

## Компоненты
- `Bazel` — билд‑система.
- `rules_apple` / `rules_swift` — правила для iOS/Swift.
- `rules_xcodeproj` — генерация `.xcodeproj`.

## Установка (верхнеуровнево)
- Подключить правила в `WORKSPACE`/`MODULE.bazel`.
- Описать таргеты (`apple_application`, `swift_library`).
- Сгенерировать проект: `bazel run //:xcodeproj` (пример цели).

## Плюсы / Минусы
- Плюсы: масштабируемость, кэш, строгие графы зависимостей, повторяемость.
- Минусы: высокий порог входа, сложная настройка правил и интеграции, требования к CI.
